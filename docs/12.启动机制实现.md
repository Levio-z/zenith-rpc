# 一、需求分析

## 1.1 为什么需要统一的启动机制？

RPC 框架涉及多个组件：注册中心、序列化器、负载均衡器、重试策略等。统一的启动机制可以：

### 问题场景

```
┌────────────────────────────────────────────────────────┐
│                    RPC 框架启动                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│   ┌──────────┐   ┌──────────┐   ┌──────────┐          │
│   │ 注册中心  │   │ 序列化器 │   │ 负载均衡 │          │
│   └────┬─────┘   └────┬─────┘   └────┬─────┘          │
│        │               │               │                │
│        └───────────────┴───────────────┘                │
│                        │                                │
│                    需要统一初始化                        │
│                        │                                │
│        ┌───────────────┴───────────────┐                │
│        │                               │                │
│   ┌────┴────┐                   ┌──────┴─────┐         │
│   │ Provider │                   │  Consumer   │         │
│   │ Bootstrap│                   │  Bootstrap  │         │
│   └─────────┘                   └────────────┘         │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 统一启动的价值

1. **简化使用**：用户只需调用一个方法即可启动框架
2. **保证顺序**：确保组件按正确顺序初始化
3. **配置管理**：统一的配置加载和管理
4. **生命周期**：统一管理资源的创建和销毁

# 二、设计方案

## 2.1 框架核心类：RpcApplication

```java
package com.zenith.zenithrpc;

import com.zenith.zenithrpc.config.RegistryConfig;
import com.zenith.zenithrpc.config.RpcConfig;
import com.zenith.zenithrpc.constant.RpcConstant;
import com.zenith.zenithrpc.registry.Registry;
import com.zenith.zenithrpc.registry.RegistryFactory;
import lombok.extern.slf4j.Slf4j;

@Slf4j
public class RpcApplication {

    private static volatile RpcConfig rpcConfig;

    /**
     * 框架初始化，支持传入自定义配置
     */
    public static void init(RpcConfig newRpcConfig) {
        rpcConfig = newRpcConfig;
        log.info("rpc init, config = {}", newRpcConfig.toString());

        // 注册中心初始化
        RegistryConfig registryConfig = rpcConfig.getRegistryConfig();
        Registry registry = RegistryFactory.getInstance(registryConfig.getRegistry());
        registry.init(registryConfig);
        log.info("registry init, config = {}", registryConfig);

        // 创建并注册 Shutdown Hook，JVM 退出时执行清理
        Runtime.getRuntime().addShutdownHook(new Thread(registry::destroy));
    }

    /**
     * 框架初始化，使用默认配置
     */
    public static void init() {
        RpcConfig rpcConfig;
        try {
            // 从配置文件加载
            rpcConfig = ConfigUtils.loadConfig(
                RpcConfig.class,
                RpcConstant.DEFAULT_CONFIG_PREFIX
            );
        } catch (Exception e) {
            log.error("rpc init error", e);
            rpcConfig = new RpcConfig();
        }
        init(rpcConfig);
    }

    /**
     * 获取 RPC 配置
     */
    public static RpcConfig getRpcConfig() {
        if (rpcConfig == null) {
            synchronized (RpcApplication.class) {
                if (rpcConfig == null) {
                    rpcConfig = new RpcConfig();
                }
            }
        }
        return rpcConfig;
    }
}
```

**设计要点**：
- **双重检查锁定**：保证配置对象的线程安全
- **配置优先级**：支持自定义配置 > 配置文件 > 默认值
- **Shutdown Hook**：JVM 退出时自动清理资源

# 三、配置加载机制

## 3.1 配置工具类

```java
package com.zenith.zenithrpc.utils;

import cn.hutool.core.util.StrUtil;
import cn.hutool.setting.dialect.Props;

/**
 * 配置工具类
 */
public class ConfigUtils {

    /**
     * 加载配置对象
     *
     * @param tClass  配置类
     * @param prefix  配置前缀
     * @return 配置对象
     */
    public static <T> T loadConfig(Class<T> tClass, String prefix) {
        return loadConfig(tClass, prefix, "");
    }

    /**
     * 加载配置对象，支持环境隔离
     *
     * @param tClass      配置类
     * @param prefix      配置前缀
     * @param environment 环境标识（如 dev, test, prod）
     * @return 配置对象
     */
    public static <T> T loadConfig(Class<T> tClass, String prefix, String environment) {
        StringBuilder configFileBuilder = new StringBuilder("application");
        if (StrUtil.isNotBlank(environment)) {
            configFileBuilder.append("-").append(environment);
        }
        configFileBuilder.append(".properties");

        // 使用 Hutool 的 Props 读取配置文件
        Props props = new Props(configFileBuilder.toString());

        // 转换为配置对象
        return props.toBean(tClass, prefix);
    }
}
```

# 四、服务提供者启动

## 4.1 ProviderBootstrap

```java
package com.zenith.zenithrpc.bootstrap;

import com.zenith.zenithrpc.RpcApplication;
import com.zenith.zenithrpc.config.RegistryConfig;
import com.zenith.zenithrpc.config.RpcConfig;
import com.zenith.zenithrpc.model.ServiceMetaInfo;
import com.zenith.zenithrpc.model.ServiceRegisterInfo;
import com.zenith.zenithrpc.registry.LocalRegistry;
import com.zenith.zenithrpc.registry.Registry;
import com.zenith.zenithrpc.registry.RegistryFactory;
import com.zenith.zenithrpc.server.tcp.VertxTcpServer;

import java.util.List;

/**
 * 服务提供者启动类
 */
public class ProviderBootstrap {

    /**
     * 初始化并启动服务提供者
     *
     * @param serviceRegisterInfoList 要注册的服务列表
     */
    public static void init(List<ServiceRegisterInfo> serviceRegisterInfoList) {
        // 1. RPC 框架初始化
        RpcApplication.init();

        // 2. 注册服务到本地注册表
        for (ServiceRegisterInfo serviceRegisterInfo : serviceRegisterInfoList) {
            String serviceName = serviceRegisterInfo.getServiceName();
            LocalRegistry.register(serviceName, serviceRegisterInfo.getImplClass());

            // 3. 注册服务到注册中心
            RpcConfig rpcConfig = RpcApplication.getRpcConfig();
            RegistryConfig registryConfig = rpcConfig.getRegistryConfig();
            Registry registry = RegistryFactory.getInstance(registryConfig.getRegistry());

            ServiceMetaInfo serviceMetaInfo = new ServiceMetaInfo();
            serviceMetaInfo.setServiceName(serviceName);
            serviceMetaInfo.setServiceHost(rpcConfig.getHost());
            serviceMetaInfo.setServicePort(rpcConfig.getServerPort());

            try {
                registry.register(serviceMetaInfo);
            } catch (Exception e) {
                throw new RuntimeException(e);
            }
        }

        // 4. 启动 TCP 服务器
        VertxTcpServer vertxTcpServer = new VertxTcpServer();
        vertxTcpServer.doStart(RpcApplication.getRpcConfig().getServerPort());
    }
}
```

## 4.2 启动流程

```
┌─────────────────────────────────────────────────────────┐
│              ProviderBootstrap.init()                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. RpcApplication.init()                              │
│     ├── 加载配置                                        │
│     ├── 初始化注册中心                                  │
│     └── 注册 Shutdown Hook                              │
│                                                         │
│  2. 注册服务到本地                                      │
│     └── LocalRegistry.register()                        │
│                                                         │
│  3. 注册服务到注册中心                                  │
│     └── Registry.register()                             │
│                                                         │
│  4. 启动 TCP 服务器                                     │
│     └── VertxTcpServer.doStart()                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 4.3 使用示例

```java
public class ProviderApplication {
    public static void main(String[] args) {
        // 定义要注册的服务
        List<ServiceRegisterInfo> services = Arrays.asList(
            ServiceRegisterInfo.builder()
                .serviceName(UserService.class.getName())
                .implClass(UserServiceImpl.class)
                .build(),
            ServiceRegisterInfo.builder()
                .serviceName(OrderService.class.getName())
                .implClass(OrderServiceImpl.class)
                .build()
        );

        // 启动服务提供者
        ProviderBootstrap.init(services);
    }
}
```

# 八、扩展思路

## 8.1 配置中心集成

```java
// 支持从配置中心（如 Nacos、Apollo）加载配置
public class ConfigCenterBootstrap {
    public static void init() {
        // 1. 初始化配置中心客户端
        ConfigCenterClient client = ConfigCenterClient.getInstance();

        // 2. 监听配置变化
        client.subscribe("rpc-config", config -> {
            // 热更新配置
            RpcApplication.updateConfig(config);
        });

        // 3. 获取初始配置
        RpcConfig rpcConfig = client.getConfig("rpc-config");
        RpcApplication.init(rpcConfig);
    }
}
```

## 8.2 优雅停机

```java
public class GracefulShutdown {
    public static void shutdown() {
        // 1. 停止接收新请求
        server.stopAccept();

        // 2. 等待现有请求处理完成
        server.awaitTermination(30, TimeUnit.SECONDS);

        // 3. 注销服务
        registry.unregister();

        // 4. 关闭资源
        resource.close();
    }
}
```

## 8.3 健康检查

```java
public class HealthCheck {
    public static boolean isHealthy() {
        // 检查注册中心连接
        if (!registry.isConnected()) {
            return false;
        }

        // 检查服务器状态
        if (!server.isRunning()) {
            return false;
        }

        return true;
    }
}
```

# 九、总结

## 9.1 启动流程

| 角色 | 启动步骤 |
|------|---------|
| Provider | 1. 框架初始化 → 2. 本地注册 → 3. 注册中心注册 → 4. 启动 TCP |
| Consumer | 1. 框架初始化 → 2. 可直接使用代理调用 |

## 9.2 关键设计

1. **统一入口**：`RpcApplication` 提供统一的初始化方法
2. **配置驱动**：支持配置文件 + 代码配置
3. **SPI 机制**：组件可插拔
4. **生命周期**：完整的创建-初始化-销毁流程

## 9.3 使用模式

```java
// 提供者
ProviderBootstrap.init(serviceList);

// 消费者
ConsumerBootstrap.init();
UserService service = ServiceProxyFactory.getProxy(UserService.class);
```
