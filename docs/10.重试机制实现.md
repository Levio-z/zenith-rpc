# 一、需求分析

## 1.1 为什么需要重试机制？

在分布式系统中，网络波动、服务短暂不可用等情况时有发生。重试机制可以在瞬时故障时自动恢复，提高系统的可用性和稳定性。

### 问题场景

```
┌─────────┐                    ┌─────────┐
│ 消费者   │  ──请求──>          │ 提供者   │
└─────────┘                    └─────────┘
     │                              │
     │                              │
     │                              X 网络抖动/服务重启
     │                              │
     │ <──超时/连接失败──           │
     │                              │
     ? 如何处理？
```

### 重试的价值

1. **容错性**：瞬时故障时自动恢复
2. **提高成功率**：临时性问题可以通过重试解决
3. **用户体验**：减少因网络波动导致的调用失败

## 1.2 重试的风险

| 风险 | 说明 | 解决方案 |
|------|------|----------|
| 雪崩效应 | 大量请求同时重试 | 限流、退避策略 |
| 幂等性问题 | 非幂等操作重复执行 | 接口幂等设计 |
| 资源浪费 | 失败请求占用资源 | 超时控制 |
| 副作用 | 重复操作产生错误结果 | 业务层面处理 |

# 二、设计方案

## 2.1 重试策略接口

```java
package com.zenith.zenithrpc.fault.retry;

import com.zenith.zenithrpc.model.RpcResponse;

import java.util.concurrent.Callable;

/**
 * 重试策略接口
 */
public interface RetryStrategy {

    /**
     * 执行重试
     *
     * @param callable 待执行的任务
     * @return RPC 响应结果
     * @throws Exception 如果执行失败
     */
    RpcResponse doRetry(Callable<RpcResponse> callable) throws Exception;
}
```

**设计说明**：
- 使用 `Callable` 封装可执行的任务
- 返回 `RpcResponse` 统一响应格式
- 异常透传，便于上层处理

## 2.2 重试关键配置

| 配置项 | 说明 |
|--------|------|
| 重试次数 | 最大重试次数 |
| 重试间隔 | 每次重试之间的等待时间 |
| 重试条件 | 什么情况下触发重试 |
| 停止策略 | 何时停止重试 |

# 三、重试策略实现

## 3.1 不重试策略

最简单的策略，调用一次就返回结果或异常。

```java
package com.zenith.zenithrpc.fault.retry;

import com.zenith.zenithrpc.model.RpcResponse;
import java.util.concurrent.Callable;

/**
 * 不重试策略
 */
public class NoRetryStrategy implements RetryStrategy {

    @Override
    public RpcResponse doRetry(Callable<RpcResponse> callable) throws Exception {
        // 直接执行，不重试
        return callable.call();
    }
}
```

**适用场景**：
- 幂等性无法保证的操作
- 对响应时间敏感的业务
- 已实现其他容错机制

## 3.2 固定间隔重试策略

每次重试之间等待固定的间隔时间。

```java
package com.zenith.zenithrpc.fault.retry;

import com.github.rholder.retry.Retryer;
import com.github.rholder.retry.RetryerBuilder;
import com.github.rholder.retry.StopStrategies;
import com.github.rholder.retry.WaitStrategies;
import com.github.rholder.retry.RetryListener;
import com.github.rholder.retry.Attempt;
import com.zenith.zenithrpc.model.RpcResponse;

import java.util.concurrent.Callable;

/**
 * 固定时间间隔重试策略
 */
public class FixedIntervalRetryStrategy implements RetryStrategy {

    @Override
    public RpcResponse doRetry(Callable<RpcResponse> callable) throws Exception {
        // 使用 Guava-Retrying 库实现重试
        Retryer<RpcResponse> retryer = RetryerBuilder.<RpcResponse>newBuilder()
            // 设置重试条件：重试所有异常
            .retryIfExceptionOfType(Exception.class)
            // 设置等待策略：每次重试等待 3 秒
            .withWaitStrategy(WaitStrategies.fixedWait(3L, TimeUnit.SECONDS))
            // 设置停止策略：最多重试 3 次
            .withStopStrategy(StopStrategies.stopAfterAttempt(3))
            // 添加重试监听器
            .withRetryListener(new RetryListener() {
                @Override
                public <V> void onRetry(Attempt<V> attempt) {
                    // 记录重试日志
                    if (attempt.getAttemptNumber() > 1) {
                        System.out.println("重试次数：" + (attempt.getAttemptNumber() - 1));
                    }
                }
            })
            .build();

        return retryer.call(callable);
    }
}
```

### Guava Retrying 核心概念

```
┌─────────────────────────────────────────────�┐
│              RetryerBuilder                 │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────────┐  ┌─────────────────┐  │
│  │  RetryIf        │  │  WaitStrategy    │  │
│  │  决定是否重试   │  │  重试间隔策略    │  │
│  └─────────────────┘  └─────────────────┘  │
│                                             │
│  ┌─────────────────┐  ┌─────────────────┐  │
│  │  StopStrategy   │  │  RetryListener  │  │
│  │  何时停止重试   │  │  重试监听器     │  │
│  └─────────────────┘  └─────────────────┘  │
│                                             │
└─────────────────────────────────────────────┘
```

### 等待策略类型

| 策略 | 说明 | 示例 |
|------|------|------|
| `fixedWait` | 固定等待 | 每次等 3 秒 |
| `randomWait` | 随机等待 | 1-3 秒随机 |
| `exponentialWait` | 指数退避 | 2^attempt 秒 |
| `fibonacciWait` | 斐波那契 | 费波那契秒数 |

### 停止策略类型

| 策略 | 说明 |
|------|------|
| `stopAfterAttempt` | 固定次数后停止 |
| `stopAfterDelay` | 超时后停止 |
| `neverStop` | 永不停止 |

## 5.2 重试策略工厂

```java
package com.zenith.zenithrpc.fault.retry;

import com.zenith.zenithrpc.serializer.SpiLoader;

/**
 * 重试策略工厂
 */
public class RetryStrategyFactory {

    static {
        // 启动时加载所有重试策略
        SpiLoader.load(RetryStrategy.class);
    }

    /**
     * 默认重试策略：不重试
     */
    private static final RetryStrategy DEFAULT_RETRY_STRATEGY =
        new NoRetryStrategy();

    /**
     * 获取重试策略实例
     *
     * @param key 重试策略 key
     * @return 重试策略实例
     */
    public static RetryStrategy getInstance(String key) {
        return SpiLoader.getInstance(RetryStrategy.class, key);
    }
}
```

# 九、总结

## 9.1 实现要点

1. **接口设计**：抽象 `RetryStrategy` 接口，支持多种策略
2. **Guava Retrying**：利用成熟库简化实现
3. **SPI 机制**：支持动态加载自定义重试策略
4. **工厂模式**：统一管理策略实例

## 9.2 使用场景

| 场景 | 推荐策略 |
|------|---------|
| 查询操作 | 固定间隔重试 |
| 写操作 | 不重试 + 幂等设计 |
| 高并发 | 指数退避 |
| 关键交易 | 有限重试 + 告警 |

## 9.3 注意事项

- 确保接口幂等性
- 设置合理的重试次数和间隔
- 监控重试次数和失败率
- 避免重试风暴
